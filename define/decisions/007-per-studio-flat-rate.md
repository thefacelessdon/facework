# Decision 007: Per-Studio Flat Rate Licensing Model

## The Question
How are agent skills licensed to studios at Beta entry?

## Resolution
Per-studio flat rate: $1K–$3K/month per studio, regardless of how many
projects they run through the protocol. Simple, predictable, non-extractive.

## Alternatives Considered
- Per-project fee: Rejected — adds tracking overhead and penalizes studios
  for using the protocol more. Per-project is a container model (you pay
  more as you grow). Flat rate is a conduit model (the more you use it,
  the more value per dollar).
- Defer to Beta: Rejected — the directional decision matters now so
  architecture doesn't create per-project tracking requirements

## Reasoning
Flat rate aligns incentives: studios succeed by running more projects
through the protocol, and Facework doesn't extract more as they scale.
This mirrors the conduit-not-container principle from the protocol itself.
Exact pricing calibrates at Beta based on market signal.

## Implications
- Architecture: No per-project metering or tracking needed in tooling
- Beta pricing: $1K–$3K/month is directional. Calibrate based on
  studio client revenue and value delivered.
- Scale: Revenue grows by adding studios, not by extracting more per studio

## Coherence Impact
High coherence. The licensing model practices the protocol's own principles.
A methodology about preventing extraction cannot have an extractive
licensing model.
