# Facework Protocol — Openness Guarantee

Version: 1.0
Date: 2026-03-23

---

## Principle

The Facework Protocol specification is open. The theory, methodology,
lifecycle phases, gate structure, and conformance criteria are publicly
available and freely referenceable. This is non-negotiable.

---

## What Is Open

1. **The Theory of Cultural Physics** — the paradigm-level framework
2. **The Coherence Operating System** — the primitive stack and engines
3. **The Facework Protocol spec** — lifecycle phases, canonical objects,
   gate criteria, conformance requirements
4. **The Build Methodology** — the seven-phase build sequence
5. **The coherence tracker** — how progress is measured

Anyone can read these, reference them, teach them, critique them, or
build with them. No permission required. No attribution required (though
appreciated).

---

## What Is Commercial

1. **The agent skills** (/fw-* tooling) — the commercial implementation
   of the protocol. Licensed, not open.
2. **Build services** — Facework running the protocol for clients. Paid.
3. **Certification** — the "Facework Protocol Conformant" mark. Earned
   through audit. Paid.
4. **Studio licensing** — the right to use the agent skills and certify
   work commercially. Paid.

---

## The Boundary

The protocol defines WHAT a coherent system looks like.
The commercial layer provides HOW to build one efficiently.

Anyone can build a protocol-conformant system without paying Facework.
The commercial layer makes it faster, more rigorous, and certifiable —
but it is never required.

---

## Why This Matters

An open protocol that can only be implemented through paid tooling is a
container disguised as a conduit. The openness guarantee prevents Facework
from becoming the extraction pattern it was designed to counter.

The test: **Can someone who has never paid Facework build a fully
protocol-conformant system using only the open spec?**

The answer must always be yes.
