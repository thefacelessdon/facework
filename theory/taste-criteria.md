# Taste Criteria

Taste is a governance function, not decoration.

## Role Of Taste

Taste decides what enters the system and what is rejected.
In Facework, taste should be explicit enough to operationalize.

## Criteria

1. **Signal Fidelity** - does this reflect the true cultural signal rather than trend mimicry?
2. **Context Integrity** - does it respect the source context and audience meaning?
3. **Compositional Clarity** - is hierarchy, rhythm, and flow legible under real use?
4. **Behavioral Coherence** - does interaction behavior match declared identity?
5. **Durability Under Scale** - does quality hold as distribution and team size grow?

## Reject Conditions

Reject outputs when they:
- optimize for novelty while breaking coherence,
- mimic aesthetics without underlying logic,
- flatten cultural context into generic style,
- improve conversion metrics by increasing extraction risk.

## Operational Use

Every major artifact should pass a lightweight taste check before launch:
- message and narrative,
- interface behavior,
- workflow design,
- and operator-facing defaults.
